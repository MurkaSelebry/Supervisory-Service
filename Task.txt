Задача 6:
 В рамках проводимых контрольно-надзорных мероприятий  появляются проблемные объекты, требующие дополнительной работы межведомственных рабочих групп по нормализации состояния объекта или принятия решения о его сносе.
При ведении рабочих групп у участников процесса возникает проблема с учетом и отслеживанием всех принятых решений и их исполнения. Теряется фокус на объектах требующих включения на рассмотрение.
 Также необходимо проверить качество исполнения работ и соблюдение сроков.
Необходимо разработать гибкий (настраиваемый по атрибутному составу) единый сервис, с возможностью добавления фотоматериалов и документов к объекту контроля (в том числе с возможность загрузки реестра XML,
 который разложится на отдельные карточки - элементы работы)), счетчиком исполнения принятых решений и формированием повестки по новым объектам. а  также в случае окончания срока исполнения поручений или его нарушения. 
Кроме того должен быть гибкий (настраиваемый функционал) по настройке  протокола принятых решений.
Основная проблема в том, что информация разрознена, требуется ручное ведение протокола и контроля исполнения протокола.
Решение:
Решением является сервис, интегрируемый в закрытую внутреннюю систему систему, который предоставляет следующие возможности:
- Работать с карточкой объекта
	- Необходимо предусмотреть  гибко настраиваемые поля (по новым атрибутам): карточка объекта должна настраиваться по разделам. 
		- Адрес
		- Кадастровый еномер
		- Площадь
		- Год постройки
		- Ответственный за обьект
		- Материал
		- Этажи
	- Предусмотреть пространство для работы конкретных пользователей с настройкой прав и возможностью добавления документов и установки контрольных даты
- Загрузка объекта
	- Ручное добавление 
	- При помощи API (отправка и принятие запроса)
	- С помощью XML
- Редактирование объекта
- Удаление объекта
- Дополнительно: Возможность отражения объекта на карте
- Работать с реестром объектов 
	- Предусмотреть возможность настройки фильтров для определения выборки 	или поиска конкретных объектов, подходящих критериям
	- Выводить реестр на одной странице с дашбордом
	- Возможность выгрузить отчет по фильтрам
- Работать с решениями
 	- При проведении рабочей группы предусмотреть возможность настраиваемых 	принятых решений по списку, установлению ответственного и срока.
	- В данную форму должны входить:формулировка срок исполнения 	ответственный
	- Также необходимо предусмотреть произвольную форму заполнения (?CanBeNull) 
	- Возможность поиска принятых/запланированных решений (пример: поиск по дате решения)
	- Возможность ввести календарь событий по разным типам объектам
	- Дополнительно: возможность формирования видеоконференций для обсуждения решений
	- Просматривать дашборд по проделанной работе и объектам, находящимся в работе
	- Возможность выгрузить отчет по дашборду
